# Protheus System Health Audit Framework

Run this checklist every 3-7 days (or after major upgrades). Score each item 1-10 and note action items.

## 1. Completeness — Is Anything Missing?
- Does every major capability (assimilation, self-improvement, task execution, value flow, etc.) have a clear home?
- Are there any TODO comments, disabled flags, or future stubs that should be promoted?
- Are core user journeys (idea -> micro-tasks -> execution -> attribution -> residuals) fully covered end-to-end?
- Missing primitives? (e.g., energy accounting, long-term memory distillation, succession protocol)

## 2. Efficiency & Resource Usage
- Surface budget behavior: Is the organism intelligently compressing/expanding based on hardware?
- Memory & CPU usage under load (shadow loops, decomposition, assimilation).
- Token burn rate vs value created — is it trending downward?
- Are there any hot paths that could be cached, profiled, or distilled into a lighter primitive?

## 3. Wiring & Architectural Integrity
- Are all organs communicating through clean primitives/profiles (no direct function calls between high-level organs)?
- Check for circular dependencies or tight coupling.
- Is data flowing correctly through passport chain, Helix manifest, and receipts?
- Duality evaluation: Is it running as the first advisory signal everywhere without blocking?

## 4. Bugs & Reliability
- Run full test suite (including new duality + task decomposition tests).
- Check for unhandled edge cases in shadow receipts over the last 24h.
- Any drift in identity anchor or constitution compliance?
- Red-team / Soldier Ant logs: Any unresolved malice detections or quarantine events?

## 5. Primitive-First Purity & Generalization Opportunities
- For every new or existing feature: Is it a primitive, a profile on a primitive, or bespoke code?
- Can any bespoke code be generalized into a new/extended primitive?
- Are there any special-case handlers that could be collapsed into Weaver or universal executor profiles?
- Check for app logic leaking into kernel (Storm-specific code in core systems).

## 6. Governance, Alignment & Duality Integration
- Does every decision surface show a duality balance score + 0-point insight?
- Is the Duality Seed influencing reasoning without overriding Heroic Echo or constitution?
- Any signs of philosophical lock-in or confirmation bias in recent self-proposals?
- Heroic Echo filtering: Are negative/self-defeating paths being starved effectively?

## 7. Security & Sovereignty (Extra Critical)
- Helix tamper checks, Soldier Ant enforcement, biometric soul token status.
- Sandbox envelope usage on all external calls and code changes.
- Any new graft/assimilation that bypassed admission checks?

## 8. Observability & Explainability
- Holo-viz clarity: Can you understand why a decision was made in <10 seconds?
- Receipt quality: Do duality scores, energy balance, and attribution show up clearly?
- Can the organism explain its own reasoning in natural language when asked?

## 9. Scalability & Hardware Agnosticism
- Surface budget test: Compress to phone-seed size and verify functionality.
- Does behavior remain identical (only capacity changes) across simulated hardware profiles?

## 10. Self-Improvement Health (Once Loops Are Live)
- Are proposed changes passing long-horizon mirror simulation?
- Is the system learning from its own loops (distilling new primitives)?
- Drift rate on core invariants (identity, constitution, duality seed).

## Usage
- Manual mode: complete every few days and store the scorecard artifact.
- Automated mode: run an audit script that outputs section scores plus prioritized actions.
