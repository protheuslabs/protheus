# Enterprise Onboarding Pack

`V4-FORT-002` onboarding summary for Operator, Platform Engineer, and External Contributor tracks.

## Prerequisites

- local clone access
- bootstrap readiness (`npm ci`, `npm run build`, `npm run test`)
- policy reading baseline (`README.md`, `CONTRIBUTING.md`, `client/docs/ONBOARDING_PLAYBOOK.md`)

## Tracks

- Operator: day-0/day-7/day-30 operational path
- Platform Engineer: day-0/day-7/day-30 implementation path
- External Contributor: day-0/day-7/day-30 collaboration path

## Safety Gates

- tests/client/docs/changelog required for merge
- no bypass of security disclosure process

## Success Criteria

- first contribution produced with complete evidence checklist
- escalation path is known before first production-affecting change
