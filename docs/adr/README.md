# Architecture Decision Records (ADR)

This directory stores architecture decisions that materially affect runtime behavior, safety, or governance.

## Process

1. Create a new ADR from `TEMPLATE.md`.
2. Assign next numeric id (`0001`, `0002`, ...).
3. Update `INDEX.md` with status and summary.
4. Link PR/commit evidence and impacted contracts.

## Status Values

- Proposed
- Accepted
- Superseded
- Deprecated
