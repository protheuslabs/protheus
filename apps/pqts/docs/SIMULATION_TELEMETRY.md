# Simulation Suite Telemetry

This layer adds deterministic, machine-readable telemetry for scenario-level simulation runs so we can measure what works and where optimization is needed.

## Entry Point

```bash
python scripts/run_simulation_suite.py \
  --markets crypto,equities,forex \
  --strategies market_making,funding_arbitrage,cross_exchange
```

## Outputs

1. `data/analytics/simulation_events.jsonl`
- Append-only event stream.
- Event types: `run_started`, `cycle_snapshot`, `run_completed`.
- Per-event metrics include:
  - order submission/fill/reject stats
  - slippage MAPE and p95 slippage
  - readiness flags
  - operational alert counts
  - promotion decision metadata

2. `data/reports/simulation_suite_<timestamp>.json`
- Batch report for the suite execution.
- Includes scenario-level results and path references.

3. `data/reports/simulation_leaderboard_<timestamp>.csv`
- Aggregated leaderboard by `(market, strategy)`.
- Columns:
  - `avg_quality_score`
  - `avg_fill_rate`
  - `avg_reject_rate`
  - `avg_slippage_mape_pct`
  - `canary_ready_rate`
  - `promote_rate`
  - `optimization_priority`

4. `data/reports/execution_drift_<timestamp>.json`
- Generated via:
  `python scripts/execution_drift_report.py --tca-db <path>`
- Quantifies paper/live parity drift per symbol@venue using:
  - slippage MAPE
  - realized/predicted slippage ratio
  - sample sufficiency thresholds

5. `data/analytics/shadow_stream_events.jsonl`
- Generated via:
  `python scripts/run_shadow_stream_worker.py --cycles 30 --sleep-seconds 1.0`
- Append-only raw parity stream with `market`, `order`, and `fill` events.

6. `data/analytics/reconciliation_incidents.jsonl`
- Generated by reconciliation daemon when internal-vs-venue mismatches breach tolerance.
- Run with:
  `python scripts/run_reconciliation_daemon.py --halt-on-mismatch`

7. `data/reports/slo_health_<timestamp>.json`
- Generated via:
  `python scripts/slo_health_report.py`
- Includes objective-level SLO evaluations and routed alerts.

8. `data/reports/error_budget_review_<timestamp>.json`
- Generated via:
  `python scripts/weekly_error_budget_review.py --window-days 7`
- Weekly aggregation of SLO breaches, burn rates, and objective status.

## Optimization Signals

- `quality_score` increases with higher fill quality and lower reject/slippage error.
- `optimization_priority` rises when quality is low, reject rate is high, or slippage error is high.
- Queue-position-aware fills now degrade fill ratio/slippage when top-of-book queue ahead is large, reducing optimistic paper assumptions.
- Stream worker health tracks uptime ratio and per-venue latency/rejection/failure rates for operational gates.
- Reconciliation daemon can engage router manual halt automatically when mismatch count exceeds policy.
- Use the leaderboard to:
  1. keep capital focused on top `avg_quality_score` scenarios,
  2. route research toward top `optimization_priority` scenarios,
  3. verify improvements by comparing new suite runs against prior telemetry.
